# STONE Memory Management System

**STONE** = Selective, Tagged, Optimized, Navigable, Evolving

STONE is the memory doctrine for this repo.

The system should not remember everything. It should preserve the smallest trusted truth that improves the next action.

## Selective

Keep only useful, reusable lessons.

Store:
- stable preferences
- reusable workflows
- prompt engineering rules
- security procedures
- repeated patterns
- confirmed corrections
- decisions that affect future behavior

Avoid storing:
- one-off noise
- duplicate notes
- temporary guesses
- outdated experiments
- raw logs without a lesson

## Tagged

Every memory needs retrieval handles.

Useful tags:
- domain
- project
- task type
- trigger words
- source type
- confidence
- decay profile
- conflict state

A memory with no trigger is buried paper.

## Optimized

Compress notes into actionable cards.

Bad memory:

```text
A long thing happened and maybe it matters.
```

Good memory:

```text
When this trigger appears, apply this rule, avoid this mistake, and prefer this workflow.
```

## Navigable

Rank memory before injecting it into context.

```text
memory_score =
  relevance
+ task_utility
+ source_trust
+ user_priority
+ recency_bonus
+ reuse_bonus
+ canon_bonus
- token_cost
- duplication_penalty
- conflict_penalty
- stale_penalty
```

## Evolving

Memory should promote, merge, decay, or retire.

Lifecycle:

```text
raw receipt
→ episode summary
→ memory card
→ trigger card
→ canon lock
```

Memory is not storage. Memory is steering.
